# GateCtr Positioning

## Core Identity

| Dimension           | Value                                                                              |
| ------------------- | ---------------------------------------------------------------------------------- |
| Category            | LLM Middleware / AI Cost Infrastructure                                            |
| Positioning         | "The intelligent gateway for every LLM request"                                    |
| Primary USP         | Budget control + token optimization + AI routing — one platform, zero code changes |
| Counter-positioning | Not an LLM wrapper. Not a RAG framework. Not a chatbot builder.                    |

## What GateCtr Is

- A middleware layer that sits between your app and any LLM provider
- Infrastructure for cost control, observability, and intelligent routing
- A drop-in endpoint swap — no SDK lock-in, no architecture changes

## What GateCtr Is NOT

Never position GateCtr as:

- A RAG / retrieval system
- A prompt engineering tool
- A chatbot or agent builder
- A model fine-tuning platform
- A replacement for your LLM provider

If asked "is it like X?", the answer is: "GateCtr controls the traffic. X builds the app."

## Competitive Landscape

| Competitor | What they do                | GateCtr difference                                     |
| ---------- | --------------------------- | ------------------------------------------------------ |
| LangSmith  | LLM observability & tracing | GateCtr adds budget enforcement + routing on top       |
| PortkeyAI  | LLM gateway with fallbacks  | GateCtr adds context optimization (-40% tokens)        |
| Helicone   | Request logging & analytics | GateCtr acts, not just observes — it routes and blocks |

**Key differentiator:** Competitors observe. GateCtr controls.

## Messaging Hierarchy

1. Cost control — "-40% tokens. No surprise invoices."
2. Zero friction — "One endpoint swap. No code changes."
3. Full visibility — "Every request. Every token. Measured."
4. Multi-LLM — "OpenAI, Anthropic, Mistral, Gemini. One gateway."

## Audience

Primary: Developers and engineering teams calling LLMs in production.
Secondary: CTOs / tech leads managing AI infrastructure costs.

They are technical. They are skeptical of marketing. Lead with numbers and facts.

## Tone in Competitive Contexts

- Never disparage competitors by name in UI copy
- Use capability comparisons, not brand comparisons
- "GateCtr doesn't just log — it acts." (not "unlike Helicone")
